# Security Policy

## Supported Versions

Security updates are currently provided for the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately by emailing:
- maintainers@your-project-domain.example

Please include:
- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested fix (optional)

## Response Timeline

- Initial acknowledgment: within 72 hours
- Triage and severity assignment: within 7 days
- Remediation target: based on severity and complexity

## Disclosure Process

1. We acknowledge your report.
2. We investigate and validate impact.
3. We prepare and test a fix.
4. We release a patch and publish advisory notes.

## Scope Notes

This project may integrate with third-party AI providers and tooling.
Vulnerabilities in external services should be reported to their respective
vendors, while integration vulnerabilities should be reported to this project.
